# 文件组织最佳实践

合理的文件组织能让你的笔记更好找、更好管理。

## 为什么要组织好

- **快速找到想要的**：不用翻半天
- **避免重复记录**：结构清晰就不会重复
- **Mory 理解更准**：结构化的内容 Mory 也更好理解
- **整理起来轻松**：定期清理不头疼

## 目录结构怎么设计

### 按领域分类

适合大多数人的基础结构：

```
笔记/
├── 工作/
│   ├── 项目/
│   ├── 日志/
│   └── 总结/
├── 学习/
│   ├── 编程/
│   ├── 英语/
│   └── 读书/
├── 生活/
│   ├── 旅行/
│   ├── 健康/
│   └── 财务/
└── 归档/
```

### PARA 方法

适合手上有多个项目的人：

```
笔记/
├── Projects/       # 正在做的项目
│   ├── 项目A/
│   └── 项目B/
├── Areas/          # 长期关注的领域
│   ├── 健康/
│   ├── 财务/
│   └── 职业发展/
├── Resources/      # 感兴趣的主题
│   ├── 技术/
│   ├── 设计/
│   └── 写作/
└── Archives/       # 已完成/不再活跃的
```

### Zettelkasten 方法

适合追求知识关联的人：

```
笔记/
├── Inbox/          # 快速捕获
├── Notes/          # 永久笔记
├── Literature/     # 文献笔记
└── Projects/       # 项目相关
```

## 命名规范

### 文件命名

**推荐格式：**

```
2024-01-15 会议记录-产品评审.md
项目A-需求文档-v2.md
《原子习惯》读书笔记.md
```

**命名原则：**

| 原则 | 说明 | 示例 |
|------|------|------|
| 日期前缀 | 方便按时间排序 | `2024-01-15 xxx.md` |
| 描述性名称 | 看名字知道是啥 | `会议记录-产品评审.md` |
| 避免特殊字符 | 兼容性好 | 不用 `/ \ : * ?` |
| 长度适中 | 不要太短或太长 | 10-30 个字符 |

### 文件夹命名

**推荐：**
```
工作/
学习/
2024年项目/
```

**避免：**
```
新建文件夹/
临时/
asdf/
```

## 内容组织

### 用模板

为常见类型的笔记建模板：

**日志模板：**
```markdown
# {{date}}

## 今天干了啥
-

## 进行中
-

## 明天打算
-

## 随手记
```

**会议记录模板：**
```markdown
# {{会议主题}}

**日期**：
**参与者**：
**会议目标**：

## 讨论内容
-

## 决议事项
-

## 待办事项
| 事项 | 负责人 | 截止日期 |
|------|--------|----------|
|      |        |          |
```

### 用好标题层级

合理用标题，让结构清晰：

```markdown
# 一级标题（文档主题）

## 二级标题（主要章节）

### 三级标题（子章节）

正文内容...
```

### 用链接关联

在笔记之间建立关联：

```markdown
这个想法来自于 [[读书笔记-原子习惯]]，
和 [[时间管理方法论]] 有关联。
```

## 定期整理

### 每天：处理 Inbox

- 快速捕获的内容移到正确位置
- 完善临时记录

### 每周：回顾和归档

- 检查本周的笔记
- 归档已完成的项目
- 删除不需要的

### 每月：优化结构

- 看看整体目录结构合不合理
- 合并相似的内容
- 更新索引


## 常见问题

### 文件夹层级太深

**问题**：层级超过 3-4 层，找东西费劲

**解决**：
- 扁平化一点
- 用命名代替层级
- 用搜索代替浏览

### 不知道放哪个文件夹

**问题**：分类纠结

**解决**：
- 先放 Inbox，回头整理
- 建一个"待分类"文件夹
- 用搜索代替精确分类

### 历史笔记太乱

**问题**：之前的笔记没组织

**解决**：
- 不用一次整理完
- 用到的时候顺手整理
- 让 Mory 帮你批量处理

## 小技巧

### 搜索是王道

不要太依赖文件夹浏览，善用搜索：

- MoryFlow 内置搜索
- 问 Mory 自然语言搜索
- 标签辅助

### 渐进式整理

不要追求完美，让组织跟着需要演进：

1. 先快速记录
2. 有空时整理
3. 定期优化结构
