# 提示词技巧

好的提示词能让 Mory 更懂你想要什么，给你更好的回复。

## 基本原则

### 1. 说具体点

**不太好的说法：**
```
帮我写个总结
```

**更好的说法：**
```
帮我总结"项目计划.md"这个笔记，
重点提取：目标、时间节点、负责人，
用表格形式，控制在 200 字以内。
```

### 2. 给点背景

**不太好的说法：**
```
给我一些建议
```

**更好的说法：**
```
我正在准备下周的英语四级考试，
已经复习了单词和语法，
但听力比较薄弱。
针对听力，给我一些复习建议。
```

### 3. 说明想要什么格式

**示例：**
```
用 Markdown 格式输出
```

```
用表格形式列出
```

```
分点列，每点不超过 50 字
```

```
生成一份可以直接存的笔记
```

## 让 Mory 读你的笔记

### 读指定文件

```
读取"工作/项目A/需求文档.md"，然后帮我...
```

### 搜索相关内容

```
在我的笔记里搜搜关于"时间管理"的内容
```

```
找出所有包含"会议"的笔记
```

### 读当前笔记

```
看看我正在编辑的笔记
```

### 批量读取

```
读取"学习/英语"文件夹下的所有笔记
```

## 让 Mory 创作内容

### 创建新笔记

```
帮我创建一个叫"读书笔记-原子习惯"的新笔记，
按这个结构组织：
- 核心观点
- 触动我的句子
- 行动清单
```

### 修改现有笔记

```
在"工作日志.md"末尾加上今天的记录
```

```
把"会议记录.md"里的要点整理成待办事项
```

### 整理和重构

```
帮我重新组织这篇笔记的结构，
让逻辑更清晰
```

## 让 Mory 搜信息

### 网络搜索

```
搜索一下 2024 年最新的 React 19 特性
```

```
帮我查查京都樱花季的最佳观赏时间
```

### 结合笔记和搜索

```
根据我的旅行偏好，
搜搜适合我的东京景点
```

```
看看我的学习笔记，
然后搜搜我可能遗漏的知识点
```

## 让 Mory 帮你规划

### 制定计划

```
帮我定一个两周的学习计划，
目标是掌握 Python 基础。
我是初学者，每天只有 1-2 小时学习时间。
```

### 分解任务

```
我要做一个网站改版项目，
帮我分成具体的执行步骤，
估算下每个步骤的工作量。
```

### 追踪进度

```
看看我的项目计划，
总结下当前进度和还没做的
```

## 高级技巧

### 角色扮演

```
假设你是一位资深的产品经理，
帮我评审这份需求文档，
指出可能的问题和改进建议。
```

### 思维链

```
帮我分析这个问题，
先列出所有可能的原因，
然后逐一排查，
最后给结论。
```

### 多轮迭代

第一轮：
```
帮我写一份项目方案的大纲
```

第二轮：
```
大纲不错，帮我展开第二部分
```

第三轮：
```
这部分再详细点，加点数据支撑
```

### 格式模板

```
按照以下模板生成内容：

## 背景
[描述问题背景]

## 目标
[明确要达成的目标]

## 方案
[具体的解决方案]

## 风险
[可能的风险和应对]
```

## 常见场景的说法

### 写作辅助

```
帮我把这段话改得更通顺：
[粘贴原文]
```

```
帮我扩写这个大纲，每个要点写 2-3 句话
```

```
帮我把这篇技术文章改成小白也能看懂的版本
```

### 学习总结

```
帮我把这章笔记整理成问答形式，方便复习
```

```
根据我的笔记，出 10 道测试题
```

### 工作效率

```
帮我把这份会议记录整理成：
1. 主要决议
2. 待办事项（带责任人）
3. 下次会议议题
```

```
帮我写一封邮件回复，
要礼貌地拒绝对方的请求，
并提供替代方案
```

### 代码相关

```
帮我解释这段代码是干嘛的
```

```
帮我重构这段代码，提高可读性
```

```
帮我写这个函数的单元测试
```

## 避免这样说

### 不要太笼统

❌ `帮我写点东西`

✅ `帮我写一份 500 字的产品介绍，突出三个核心功能`

### 不要假设 Mory 知道背景

❌ `继续上次的`

✅ `上次我们讨论了项目 A 的需求，请继续帮我完善技术方案部分`

### 不要一次要求太多

❌ `帮我写方案、做表格、发邮件、安排会议`

✅ 分步来，每次聚焦一件事

## 持续改进

1. **存好用的提示词**：效果好的记下来复用
2. **分析失败的对话**：Mory 理解错了，想想哪里没说清楚
3. **换换说法**：同样的需求，换个说法可能效果更好
