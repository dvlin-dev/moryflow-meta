# 用 MoryFlow 写工作总结

让 Mory 帮你把日常工作记录变成专业的工作总结。

## 适用场景

要写工作总结（周报、月报、年终总结），想要：

- 不漏重要工作内容
- 条理清晰、重点突出
- 快速生成专业格式
- 省时间

## 准备工作

### 1. 养成记录习惯

平时工作中，随手记一下。不用写得很完整，几个关键词就行。

**工作目录结构：**
```
工作/
├── 日志/
│   ├── 2024-01-15.md
│   ├── 2024-01-16.md
│   └── ...
├── 项目/
│   ├── 项目A/
│   └── 项目B/
└── 总结/
    ├── 周报/
    └── 月报/
```

### 2. 日志模板

**日志示例（2024-01-15.md）：**
```markdown
# 2024-01-15 周一

## 今天干了啥
- 完成用户管理模块的代码review
- 修了3个bug（#123, #124, #125）
- 参加产品需求评审会

## 进行中
- 订单系统重构，进度60%

## 遇到的问题
- 数据库查询性能问题，需要优化

## 明天打算
- 继续订单系统重构
- 性能优化方案调研
```

## 更多技巧

### 对比分析

```
对比下我这个月和上个月的工作产出
```

### 提取亮点

```
从这周工作里找出最值得汇报的3个亮点
```

### 格式转换

```
把周报转成 PPT 大纲格式
```

## 经验总结

1. **每天记一点**：哪怕只是几个关键词，也比什么都不记强
2. **每周整理**：别等年底抓瞎
3. **用数字说话**：量化你的工作价值
4. **突出价值**：重点不是做了什么，而是产生了什么效果
