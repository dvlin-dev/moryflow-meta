# Moryflow 官网

Moryflow 是一个本地优先的 AI Agent 产品，会思考、会规划、会不断进化。这是它的官网前端项目。

---

## 📖 项目背景

### 产品定位
- **核心身份**：Mory 不是工具，是"会思考的伙伴"
- **核心能力**：
  - 🧠 **记得你的一切**：长期记忆，用得越多越懂你
  - 🌐 **能找最新信息**：联网搜索，获取实时资讯
  - ✍️ **会帮你创作**：生成内容，制定计划
  - ✨ **更多可能...**：持续进化中，未来能力不断增加

### 设计目标
- 避免技术化表述，使用生活化场景（准备考试、策划活动、旅行规划）
- 强调"长期记忆"和"个性化"，而非冷冰冰的工具列表
- 保持未来扩展性，不把能力"限制死"

---

## 🎨 设计喜好与原则

### ✅ 喜欢的设计风格
- **玻璃磨砂质感**：`backdrop-blur`、半透明白色背景（95-99%）
- **克制的高级感**：不过度透明（避免"廉价感"）、不浮夸
- **温暖色调**：米白底色（`#F7F7F5`）+ 暖橙主色（`#FF9F1C`）
- **大圆角**：24px 统一圆角（Arc 浏览器风格）
- **精细阴影**：双层/三层阴影系统，创造深度但不生硬
- **微妙渐变**：白色 → 浅橙色，增加层次感
- **分层清晰**：卡片和背景要有明显区分，但不过度分割

### ❌ 禁止的设计元素
- **蓝紫渐变**：禁用所有蓝色+紫色的渐变组合
- **阴影/drop-shadow**：禁用传统粗重阴影（改用精细自定义阴影）
- **单调卡片式布局**：避免缺乏个性的网格堆砌
- **过度透明**：70-80% 透明度会显得"轻飘飘"
- **GitHub 元素**：不显示任何 GitHub 相关内容
- **技术术语**：避免 read/write/grep 等工具名，改用场景描述

### 🎯 透明度体系
```
98-99% ━━━ 主要卡片（Hero、Capabilities）
95-96% ━━━ 次要卡片（Download、底部总结框）
90-95% ━━━ 装饰元素（图标容器、徽章）
85%    ━━━ 输入框（带浅橙色背景）
60-70% ━━━ 未来暗示（"更多可能..."虚化卡片）
```

### 🌈 阴影系统
```css
/* 主要卡片 - 双层阴影 */
shadow-[0_8px_32px_0_rgba(0,0,0,0.08),0_2px_8px_0_rgba(0,0,0,0.04)]

/* 悬停时 - 三层阴影 */
hover:shadow-[0_16px_48px_0_rgba(0,0,0,0.12),0_4px_12px_0_rgba(0,0,0,0.06)]

/* 色彩融合层 */
shadow-gray-200/50  /* 中性稳重 */
shadow-orange-100/50 /* 温暖品牌色 */
```

### 📐 字体与排版
- **标题**：Merriweather（衬线，优雅）
- **正文**：Inter（无衬线，清晰）
- **圆角**：统一 24px（`rounded-3xl`）
- **间距**：宽松留白，避免拥挤

## 🏗️ 技术架构

### 技术栈
- **框架**：React 19 + TypeScript + Vite
- **路由**：React Router v7 (Data API)
- **样式**：Tailwind CSS（自定义主题）
- **动画**：Framer Motion
- **图标**：Lucide React
- **包管理**：pnpm

## 📝 设计决策记录

### 为什么用"会思考的伙伴"而非"AI 工具"？
- 用户不关心技术细节（read/write/grep）
- 他们关心的是：Mory 能为他们做什么
- "伙伴"比"工具"更有温度和个性

### 为什么强调"长期记忆"？
- 这是 Mory 的核心差异化（vs ChatGPT 等）
- "用得越多越懂你"建立长期关系
- 连接记忆与隐私保护（"只记得你"）

### 为什么留"更多可能..."虚化卡片？
- 避免把能力"限制死"
- 暗示持续进化，创造期待感
- 虚线边框 + 低透明度 = 视觉上的"未来感"

### 为什么用 99% 白色而非纯白卡片？
- 保留 1% 磨砂质感
- 避免和纯白背景完全一样
- 配合 `backdrop-blur` 创造空间层次

### 为什么用双层/三层阴影？
- 单层阴影：只有深度，缺乏融合
- 双层阴影：深度 + 色彩（灰色/橙色）
- 三层阴影（悬停）：强化交互反馈
